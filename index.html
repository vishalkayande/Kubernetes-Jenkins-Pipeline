Technologies Used

Git & GitHub Jenkins Docker Docker Hub Kubernetes Nginx Linux (VMs)

System Architecture

This live web application is running inside a Kubernetes Pod. Code changes pushed to GitHub automatically trigger a build, containerization, and rollout to the cluster.

Pipeline Stages

1. Source Control

GitHub detects a push event and sends a webhook payload to the Jenkins VM.

2. Integration

Jenkins clones the repository, builds the Docker image, and applies the dynamic build tag.

3. Registry Push

Jenkins securely authenticates with Docker Hub and pushes the versioned image.

4. K8s Deployment

Jenkins executes kubectl commands to apply the new container rollout to the cluster.
