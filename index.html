Automated CI/CD Pipeline

Engineered and Deployed by Vishal Kayande

System Architecture

This live web application is running inside a Kubernetes Pod. Code changes pushed to GitHub automatically trigger a build, containerization, and rollout to the cluster.

Pipeline Stages

1. Source Control

GitHub webhook detects a push event and sends a JSON payload to the Jenkins VM.

2. Integration

Jenkins clones the repository, builds the Docker image, and applies the BUILD_NUMBER as the tag.

3. Registry Push

Jenkins securely authenticates with Docker Hub and pushes the versioned image.

4. K8s Deployment

Jenkins updates the deployment.yaml with the new tag and applies it to the Kubernetes cluster.

Status: Live Container • Managed via Kubernetes NodePort
